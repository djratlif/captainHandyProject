# CaptainHandy Comic Generator

This project contains the training and inference pipeline designed to generate a daily comic for the character **CaptainHandy**. 

The AI agent is powered by a custom **LoRA (Low-Rank Adaptation)** model, which has been fine-tuned on top of **Flux.1 [dev]** using Replicate's cloud API.

## Project Structure

* `trainImgs/` - Contains the 25 target drawing images used to teach the AI what CaptainHandy looks like.
* `train.py` - The automation script that packages the training images into a `.zip` file, connects to the Replicate API, generates a private destination model, and kicks off the 15-20 minute fine-tuning process.
* `generate.py` - The inference script used to query the trained `djratlif/captainhandy-comic` model and generate new custom comic panels based on text prompts.
* `test_comic_panel.png` - Example output generated by the model.

## Setup & Requirements

### 1. Install Dependencies
You need the `replicate` and `requests` python libraries to run these scripts:
```bash
pip install replicate requests
```

### 2. Authentication
Both the training and generation scripts rely on the Replicate API. You must have a valid Replicate API token with billing enabled.

Set your token as an environment variable before running either script:
```bash
export REPLICATE_API_TOKEN='r8_your_token_here'
```

## Usage

### Generating Images
To generate a new comic panel, open `generate.py` and modify the `test_prompt` variable. To guarantee that CaptainHandy retains his classic aesthetic, ensure your prompt explicitly requests the style, such as:
> *"A simple minimalist black and white line drawing, uncolored, sketch style..."*

Then run:
```bash
python generate.py
```

### Retraining the Model
If you add more images to `trainImgs/` and want to retrain the character:
```bash
python train.py
```
*(Note: Replicate generally charges ~$1.00 USD per Flux LoRA training run).*
